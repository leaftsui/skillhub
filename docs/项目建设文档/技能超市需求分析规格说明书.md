# 技能超市需求分析规格说明书

# 1. 项目背景

## 1.1 业务背景与痛点

近年来，大模型技术快速发展，人工智能正逐步从知识问答向任务执行演进。对于银行科技研发而言，AI真正能够提升项目建设效率的关键，并不仅仅依赖模型能力，更需要将组织内部经过实践验证的工作经验沉淀为标准化、可复用、可持续演进的能力资产。

目前，我行知识管理系统建设工作已经启动，并逐步开展制度文档、产品文档、项目资料等知识资产的统一管理，为员工提供知识查询和学习能力。知识管理系统主要解决的是"知识是什么（What）"的问题，能够帮助员工快速获取制度规范、产品说明和历史经验，却无法直接回答"工作应该怎么做（How）"的问题。

在科技项目建设过程中，从需求分析、系统设计、代码开发、测试验证到生产投产，每个阶段都积累了大量优秀实践，例如需求分析模板、设计规范、Prompt模板、自动化脚本、测试检查清单以及投产规范等。这些经验多数分散在个人电脑、项目目录或团队内部，缺乏统一的沉淀、治理和共享机制，导致优秀经验难以复制，新成员培养成本较高，同类问题在不同项目中重复出现，整体研发效率和交付质量受到影响。

另一方面，随着Agent技术的发展，仅依赖知识库已无法满足AI参与项目建设的需求。AI不仅需要获取知识，更需要能够理解具体的执行步骤、输入输出要求、工具调用方式以及质量检查标准，才能真正参与需求分析、代码开发、测试辅助等实际工作。因此，需要构建面向项目建设全过程的标准化技能体系，将个人经验沉淀为组织能力，为员工和AI提供统一的执行能力支撑。

基于以上背景，拟建设银行内部Agent Skills技能超市，作为知识管理系统之上的实践执行平台，围绕科技研发全过程沉淀最佳实践，实现技能资产化、标准化、平台化管理，为后续AI Agent能力建设提供统一的技能底座。

---

## 1.2 建设目标

Agent Skills技能超市定位于知识管理系统的实践延伸层，重点解决科技研发过程中"如何完成工作"的问题。

本期项目以科技研发领域作为试点，围绕需求分析、系统设计、代码开发、测试验证、生产投产等研发全过程，建设统一的技能管理平台，将分散在个人和团队中的优秀实践沉淀为标准化技能包，逐步形成企业级研发能力资产。

技能包不仅包括Prompt、模板、自动化脚本等执行资源，还应定义适用场景、输入输出规范、执行步骤、工具调用方式、质量检查项以及关联知识，为员工提供标准化工作指导，同时支持AI Agent直接调用，实现人与AI共享同一套执行能力。

系统将建立覆盖技能注册、审核、发布、安装、使用、反馈、版本管理、持续优化等全过程的技能生命周期管理机制，确保技能能够持续演进、持续优化，不断沉淀组织最佳实践。同时，通过技能运营分析能力，对技能安装情况、调用次数、覆盖项目、节约工时等指标进行统计分析，量化技能产生的业务价值，为技能建设提供持续运营依据。

---

## 1.3 核心价值

Agent Skills技能超市的建设，不仅能够实现科技研发经验的统一沉淀，更重要的是推动知识向能力转化，形成组织级研发能力资产。

对于员工而言，技能超市能够提供标准化、可直接复用的工作能力，降低学习成本，提高项目交付效率，使优秀实践能够快速复制到不同团队和不同项目中。

对于组织而言，技能超市实现了研发过程的标准化管理，统一各阶段输入输出规范，降低沟通成本，提高跨团队协作效率，并通过技能生命周期管理不断沉淀组织能力。

对于知识管理系统而言，技能超市与知识管理系统形成互补关系。知识管理系统负责沉淀制度规范、产品文档等知识资产，解决"是什么"的问题；技能超市引用知识作为执行依据，解决"怎么做"的问题，实现知识支撑技能、技能促进知识应用的双向联动。

对于未来AI建设而言，标准化技能包天然具备AI可调用能力，为后续Agent自动执行研发任务提供统一能力接口，推动AI从知识问答逐步向实际工作执行演进。

---

# 2. 系统总体架构

## 2.1 架构概览

Agent Skills技能超市采用平台化架构设计，整体由技能管理层、技能运行层、知识支撑层和数据存储层组成。

技能管理层负责技能的注册、审核、发布、安装、权限管理、版本管理以及运营分析等核心业务，实现技能资产全生命周期管理。

技能运行层负责技能包解析、Prompt管理、模板管理、工具调用以及技能执行，为员工和AI Agent提供统一的技能运行能力。

知识支撑层依托现有知识管理系统，为技能提供制度依据、产品文档、设计规范等知识引用能力。技能包本身不存储知识内容，而是通过Reference机制关联知识管理系统，实现知识与技能解耦。

数据存储层负责技能元数据、版本信息、安装记录、调用日志、运营数据等统一管理，为统计分析和持续运营提供数据支撑。

整个系统采用模块化设计，各功能模块相互独立，便于后续扩展Workflow、Agent编排等高级能力。

---

## 2.2 存储分层设计

系统采用分层存储方式，对技能运行过程中产生的不同类型数据进行分类管理。

技能元数据用于存储技能基本信息，包括技能名称、分类、标签、适用场景、作者、部门、创建时间等属性；技能包存储技能具体内容，包括Prompt、模板、脚本、执行步骤、输入输出规范、工具配置等；版本数据用于管理技能历史版本及升级记录，支持版本比较、历史回溯和版本回滚；运营数据记录技能安装情况、调用次数、覆盖项目、用户评价、优化建议等信息，为技能持续优化提供依据；知识引用信息记录技能关联的知识资源，实现知识管理系统与技能超市之间的联动。

通过数据分层设计，实现技能内容、运行数据和知识引用的解耦，提高系统扩展性和维护效率。

---

## 2.3 数据流向

技能超市围绕技能生命周期构建完整的数据流转过程。技能创建者完成技能编写后提交平台审核，审核通过后正式发布至技能超市。用户可根据实际工作需要安装所需技能，安装后的技能可在实际项目中直接使用。系统持续采集技能使用情况、用户反馈和优化建议，支持技能持续迭代升级，形成经验沉淀、持续优化、价值提升的闭环。

与此同时，技能在执行过程中可引用知识管理系统中的制度文档、设计规范和业务知识作为理论依据，实现技能执行过程可追溯、可解释，确保最佳实践符合企业规范要求。

---

# 3. 技能领域模型

Agent Skills技能超市围绕技能资产建立统一领域模型，技能作为系统核心对象，由技能基本信息、技能内容、技能版本、技能权限、技能标签、知识引用、安装记录、使用记录和评价反馈等多个对象共同组成完整的技能资产。

其中，技能包作为技能的具体实现载体，封装Prompt、模板、脚本、工具配置、执行流程、输入输出规范以及质量检查项等内容；版本对象负责管理技能演进过程；权限对象负责控制技能可见范围及安装权限；知识引用对象负责关联知识管理系统中的制度文档、产品资料及设计规范；使用记录和评价反馈对象负责采集技能实际运行数据，为技能持续优化提供数据支撑。

整个领域模型围绕技能生命周期设计，实现技能资产标准化、规范化和可持续演进。

---

# 4. 功能需求——MVP阶段

MVP阶段重点聚焦科技研发领域，以满足技能资产管理和共享复用为核心目标，优先建设技能注册、审核、发布、安装、权限管理、版本管理、知识引用、统计分析等基础能力。

系统支持用户创建标准化技能包，对技能进行分类管理和标签管理，并支持技能审核发布流程，确保进入技能超市的技能符合企业规范要求。系统支持按部门、角色、项目组等维度控制技能访问权限，实现不同范围内的技能共享。

技能发布后，用户可根据实际工作需要安装和使用技能，平台记录技能安装及调用情况，并支持技能版本升级、历史版本查看及版本回滚，保证技能持续迭代。

技能支持关联知识管理系统中的知识资源，在技能说明及Reference中引用制度规范、设计文档等内容，实现知识与技能联动。平台提供技能搜索、分类浏览、标签检索等能力，方便用户快速查找所需技能。

同时，系统建设技能运营分析能力，对技能数量、安装人数、调用次数、覆盖项目、热门技能、优秀贡献者等指标进行统计分析，直观展示技能建设成果和业务价值。

---

# 5. 功能需求——二期规划

二期将在MVP基础上进一步完善技能智能化能力，引入技能编排、AI自动生成技能、技能推荐等功能。

系统支持多个技能按照研发流程进行组合编排，形成完整的工作流，实现需求分析、设计、开发、测试、投产等多个技能之间的自动衔接。支持基于项目总结自动提取最佳实践，由AI辅助生成技能初稿，降低技能建设成本。

同时，平台根据用户岗位、项目类型及历史使用情况，智能推荐相关技能，提高技能复用率，并结合技能反馈自动优化Prompt、模板及执行流程，实现技能持续演进。

---

# 6. 功能需求——长期目标（愿景）

长期来看，Agent Skills技能超市将逐步从科技研发领域扩展至业务、运营、风控、办公等更多业务场景，形成覆盖全行的企业级技能平台。

未来，平台将构建统一的企业技能标准，实现知识、技能、Workflow、Agent之间的深度融合，使技能成为组织能力的统一载体。员工和AI Agent共享同一套技能体系，通过技能不断沉淀组织经验，推动组织能力持续积累，最终形成面向全行的企业级Agent能力底座。

---

# 7. 非功能需求

系统应具备良好的性能、安全性、稳定性和扩展性，支持企业级用户并发访问，满足技能快速检索和安装需求。平台应建立完善的权限控制、操作审计和版本管理机制，保证技能资产安全可靠。系统需支持模块化扩展，为未来接入Agent、Workflow、MCP工具以及更多业务场景预留接口能力，并能够与知识管理系统保持良好的集成能力。

---

# 8. 界面需求（参考原型）

系统主要包括技能超市首页、技能详情页、技能创建页面、技能审核页面、技能安装管理页面、技能版本管理页面、技能运营分析页面以及个人技能中心等功能界面。整体设计应遵循简洁、统一、易操作的原则，突出技能浏览、技能安装和技能价值展示能力，为用户提供良好的使用体验。

---

# 9. 数据模型设计（初步）

系统围绕技能资产建立统一的数据模型，主要包括技能基本信息、技能包、技能版本、技能分类、技能标签、技能权限、知识引用、安装记录、调用记录、评价反馈和运营统计等核心实体。各实体之间通过统一标识进行关联，实现技能全生命周期的数据管理，并为后续统计分析、智能推荐和AI调用提供统一的数据基础。

---

# 10. 待研讨事项

当前仍有部分设计内容需要进一步讨论和明确，包括技能包标准规范、技能依赖机制、技能版本升级策略、知识引用方式、技能审核流程、权限粒度划分、AI生成技能的审核机制、技能运营评价指标以及未来Workflow和Agent编排能力的建设方案。此外，还需进一步确定技能超市与知识管理系统之间的数据同步机制及接口规范，确保两个平台既保持职责边界清晰，又能够形成统一的组织能力建设体系。
