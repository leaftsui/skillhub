## 一、代码库现状总览

SkillHub 开源项目已实现了大部分 MVP 和核心治理能力，以下是对照项目计划的三阶段评估：

| 阶段 | 计划内容 | 实现状态 | 关键差距 |
|------|----------|----------|----------|
| **W1-W3** MVP基础闭环 | 技能CRUD、分类标签、检索、包结构、安装 | **90% 已实现** | 服务端用户安装记录缺失 |
| **W4-W7** 核心业务深化 | 审核流、版本管理、权限、知识引用、统计分析 | **80% 已实现** | 知识引用无服务端实体、无分析看板 |
| **W8-W10** 技能实验室+试点 | 技能实验室、推荐系统、知识系统集成、性能优化 | **40% 已实现** | 实验室、推荐系统完全缺失 |

  ```java
  package com.iflytek.skillhub.domain.skill;

  import jakarta.persistence.*;
  import java.time.Instant;

  @Entity
  @Table(name = "skill_installation")
  public class SkillInstallation {

      @Id
      @GeneratedValue(strategy = GenerationType.IDENTITY)
      private Long id;

      @Column(name = "skill_id", nullable = false)
      private Long skillId;

      @Column(name = "skill_version_id", nullable = false)
      private Long skillVersionId;

      @Column(name = "user_id", nullable = false, length = 128)
      private String userId;

      @Column(name = "installed_at", nullable = false)
      private Instant installedAt;

      @Column(name = "uninstalled_at")
      private Instant uninstalledAt;

      protected SkillInstallation() {}

      public SkillInstallation(Long skillId, Long skillVersionId, String userId) {
          this.skillId = skillId;
          this.skillVersionId = skillVersionId;
          this.userId = userId;
          this.installedAt = Instant.now();
      }

      public void uninstall() {
          this.uninstalledAt = Instant.now();
      }

      public boolean isActive() { return uninstalledAt == null; }

      // getters
  }
  ```

### 关键设计决策

1. **安装记录不采用客户端上报**：服务端通过 API 调用直接写入 `skill_installation` 表，保证数据准确
2. **知识引用独立于技能包**：引用数据存储在关系表中而非技能包 ZIP 内，方便独立更新和搜索
3. **实验室草稿与正式技能分离**：`lab_skill_draft` 独立于 `skill_version`，草稿经发布流程才转为正式技能
4. **推荐系统先规则后模型**：W8 实现基于标签和热度的规则推荐，W10 视数据量决定是否引入协同过滤
5. **分析看板数据不要求实时**：每日汇总表 `skill_installation_daily` 定时刷新，避免高频聚合查询压力
